# Star Freight

你曾是一名军用飞行员。然后，你成为了一个耻辱。现在，你是一名船长，拥有一艘状况不佳的飞船，没有任何影响力，并且所处的星系在你的到来之前就已经在运转。

**Star Freight** 是一款以文字为基础的战术商贸角色扮演游戏，讲述了在政治分裂的星系中运输货物的故事。 包含五个文明，一个经济体系，以及四个真理，这些真理将确保你无法重复第二次人生。

---

## 游戏

你会在不同的港口停靠，每个港口的文化都会影响你的谈判方式。 你会选择不同的航线，不同的地形会影响你的战斗方式。 你会雇佣船员，他们会改变你能做什么，你能访问什么，以及你欠什么。 你会接受看似简单的合同，但当文件出现问题、物资短缺导致价格上涨，或者货物被发现是证据时，情况就会变得复杂。

你不是被选中的人。你是一名努力维持飞船运转的船长，而机构正在没收货物，物资短缺正在改变航线，船员带来了各种义务，而危险的真相则在日常工作中浮出水面。

每一次航行都将塑造你成为一个不同的船长。 这不是因为你选择了某个职业，而是因为你的船员、航线、风险和声誉塑造了你。

## 它与众不同的原因

大多数角色扮演游戏将各种系统并列呈现。 Star Freight 则让这些系统相互关联。

**船员是约束力。** Thal 不仅仅是一个+10%的维修加成。 Thal 是*你*之所以能够停靠在 Keth 港口的原因，*你*的飞船之所以拥有紧急维修能力的原因，以及*你*之所以注意到医疗货物与季节不符的原因。 失去 Thal，三个系统就会同时失去功能。

**战斗是一种战役事件。** 你不会进入一场战斗然后立即退出。 胜利会带来回收的信用和派系声望。 撤退会让你损失货物，并让你在其他派系中获得“逃兵”的声誉。 失败意味着货物被没收、船员受伤，以及飞船以高昂的价格勉强到达最近的港口。 每一个结果都会改变你下一次的贸易决策。

**文化是社会逻辑。** Keth 人不仅仅是价格不同。 他们拥有一个季节性的生物日历，这个日历会改变送礼的含义，什么时候提出交易会被视为一种侮辱，以及什么时候外来者会被限制在外部区域。 Veshan 人不仅仅是战斗。 他们会挑战，而拒绝比失败更糟糕。 知识不是一个代码，而是观察周围的环境。

**调查是从生活中诞生的。** 你之所以注意到医疗货物与季节不符，是因为你一直在运输它。 你之所以发现清单上的差异，是因为你打捞了一艘残骸。 你之所以能够理解其中的模式，是因为你的 Keth 船员记得她欠同一家供应商的债务。 阴谋不会主动暴露。 你会在完成工作时偶然发现它。

## 世界

五个文明共享一个名为“阈”的星系。

**地球联邦 (The Terran Compact)**：官僚主义的人类政府。 他们让你蒙上耻辱。 重新获得他们的信任需要许可证、耐心和放下自尊。 市场安全，利润微薄，文件繁琐。

**Keth 共同体 (The Keth Communion)**：一个遵循生物日历的节肢动物集体。 他们耐心、观察力敏锐，一旦受到冒犯，就会造成毁灭性打击。 如果你了解季节，他们将提供系统内最好的利润。 如果你不了解，你的声誉会迅速崩溃。

**Veshan 诸侯国 (The Veshan Principalities)**：痴迷于荣誉和债务的爬行动物封建家族。 他们会正式战斗，直接交易，并且记住一切。 债务账簿不仅仅是装饰，它是筹码。

**Orryn 流浪者 (The Orryn Drift)**：一个移动的经纪人文明。 他们秉持中立政策，但设计上是盈利的。 他们与所有人交易，了解一切，并对所有服务都收取费用。 避免与他们合作可以节省资金。 失去他们的信任会付出更大的代价。

**黑曜石边境 (The Sable Reach)** — 海盗势力、寻觅古代科技的拾荒者，以及“联邦”希望能够忘记的人。这里没有法律，没有习俗，没有退款。这是整个系统中风险最高、回报也最高的区域。

## 三种船长类型

同一个世界。三种不同的处境。

**守法船长 (Relief captain)。** 遵守运输纪律，基于信任的访问权限，公开的后果。你确保船员有食物，但也会因为遵守规则而陷入困境。你害怕延误和运力不足。

**灰色走私者 (Gray runner)。** 利用规则漏洞，利用时机，承担机构风险。你通过保持难以捉摸来赚钱。你害怕被没收和暴露。

**荣誉船长 (Honor captain)。** 直接对抗，处理内部政治，声誉波动。你通过面对面解决问题。你害怕事态升级和缺乏支持。

这些不是职业。而是你的选择塑造了你。

## 快速开始

```bash
# Clone and install
git clone https://github.com/mcp-tool-shop-org/star-freight.git
cd star-freight
pip install -e ".[tui]"

# Start a new game
starfreight new "Captain Nyx" --type merchant
starfreight tui
```

**操作:** `D` 仪表盘 | `C` 船员 | `R` 航线 | `M` 市场 | `T` 空间站 | `J` 日志 | `F` 派系 | `B` 购买 | `S` 出售 | `G` 航行 | `A` 推进

## 当前状态

《星际货运》是一款经过验证的产品，而不是一个设计概念。

| | 数量 |
|---|---|
| 空间站 | 9 |
| 星际航道 | 14 |
| 贸易商品 | 18 |
| 船员 | 5 + 玩家 |
| 合同 | 9 |
| 遭遇 | 6 |
| 调查线索 | 4 |
| 测试通过 | 2,200+ |

核心功能已通过所有三个验证标准：黄金路径（持续的船长生涯）、遭遇（三个具有不同游戏状态的分支）、经济（压力持续存在，不会崩溃）。

已发布三个扩展包：工作生活（展现人性）、家族、审计和没收（机构压力），以及短缺、制裁和护航（受控的稀缺）。

船长发展路径的多样性已得到验证：三种不同的立场会产生不同的路线、不同的贸易组合、不同的战斗风格、不同的失败模式，以及不同的船长身份。

## 技术

Python 3.11+。 丰富的文本用户界面。 船员绑定，网格战斗，文化知识，调查和游戏流程编排。 2200+ 个测试。

不依赖任何外部 AI。 不使用云服务。 运行在您的机器上。

**安全模型：** 《星际货运》是一款单人离线游戏。它只访问本地存档文件。它不访问网络，不收集遥测数据，不存储凭据，也不需要用户帐户。 依赖项包括 Typer、Rich 和 Textual，所有这些都经过良好维护，没有原生代码。 详细信息请参阅 [SECURITY.md](SECURITY.md)。

## 基本原则

当不确定接下来应该构建什么时：

- 它是否强化了四大原则之一？
- 它是否提升了船长的人生体验？
- 它是否会促使玩家做出选择并感受到其中的意义？

如果不是，可以暂时搁置。
